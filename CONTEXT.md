# Beater — domain language

Maze chase game with a rhythm hook. Diverges from OG Pac-Man where called out. ADRs in [docs/adr/](docs/adr/).

## Terms

- **Game state** — top-level state of a single run, owned by `Game`. One of `ready` (level intro, only at level start — not on respawn), `playing`, `paused`, `dying`, `level_complete`, `game_over`. Transitions: `ready → playing`, `playing → dying`, `dying → playing` (respawn) or `dying → game_over` (no lives left), `playing → level_complete → ready` (next level), `playing ↔ paused` (only from `playing`). Replaces the former `@level_complete` boolean.

- **Scene** — top-level mode above `Game state`, owned by `main.rb`. One of `title`, `playing` (delegates to a live `Game`), `settings`, `credits`, `instructions`, `jukebox`, `highscores`. Scene swaps go through a single `request_scene(name)` mechanism that generalises the previous `request_game_reset` — still the only full-rebuild path. `Game` instance exists only in scene `playing`; other scenes own no `Game`. Transitions use a shared ~15-frame fade-to-black with synchronised audio duck (reuses `DEATH_DUCK_GAIN_SCALE` envelope). See [ADR-0012](docs/adr/0012-scene-transition-request.md).

- **Title** — `Scene` shown at app start and after exit-to-title. Hosts game logo, tagline ("Maze. Beat. Repeat."), play button, settings link, credits link, social links (open via `$gtk.openurl`). Looping low-volume music bed. Mouse + keyboard + gamepad navigation.

- **Credits** — `Scene` reachable from title. Plain text list: design/code/music by Andrea D'Amico (kc00l @ Fifth Layer Studio); engine DragonRuby GTK; reference *The Pac-Man Dossier* by Jamey Pittman; inspired by Pac-Man (Namco, 1980) and Wizard of Wor (Midway, 1980). Esc / B / mouse-back returns to title.

- **Pause** — `paused` `Game state`. Entered from `playing` via Esc/Start. World sim, phase scheduler, and `Play clock` all frozen; music keeps playing low (ducked via `DEATH_DUCK_GAIN_SCALE`) — `BeatClock` is a pure function of `args.state.tick_count` so audio stays in phase across pause without resync logic. Menu items: resume, settings, exit to title.

- **Jukebox** — `Scene` reachable from title. Music mixer that lets the player drive each `Track`'s completion ratio in real time via vertical faders, with per-track mute/solo and a side panel that auditions every SFX. Built on the same `Audio::Manager` the live game uses, so what you hear in the jukebox is exactly what the mix produces at that progression. Doubles as a tuning tool for `LevelConfig` / `TRACK_CONFIGS`. Esc / Q exits back to title.

- **Settings** — `Scene` reachable from title and from pause. MVP knobs: master/music/sfx volume, fullscreen toggle, reduced-flash accessibility toggle, read-only controls list. Persisted to `settings.txt` (`$gtk.serialize_state`) on settings-screen exit, not per-edit. Missing file on first run → silent defaults (master 0.8 / music 0.7 / sfx 0.9, fullscreen off, reduced-flash off). Slider edits live-preview a 1-shot SFX at the new gain. Settings opened from pause returns to the paused `Game`, not the title, via a `@scene_return_to` slot.

- **Highscore table** — top-10 local leaderboard stored in `highscores.txt` (`$gtk.serialize_state`), schema `version: 1`. Each entry: `score`, `level_reached`, `time_seconds` (see `Play clock`), `initials` (3-char arcade entry), `date`. Shown on the UI5 game-over screen; new qualifying score prompts arcade-rotor initials entry (3 slots, up/down cycles A-Z, left/right switches slot, enter confirms — keyboard/gamepad/mouse). Missing file on first run → seeded with placeholder entries for visual interest. See [ADR-0013](docs/adr/0013-play-clock-pause-excluded.md).

- **Play clock** — run-duration counter for highscore display, in ticks. Incremented exactly once per `tick_playing` (60 ticks = 1 second). Does **not** advance during `ready`, `paused`, `dying`, `level_complete`, or `game_over`. Reset to 0 on full `Game` rebuild. See [ADR-0013](docs/adr/0013-play-clock-pause-excluded.md).

- **Life** — integer count of player retries in a run. Starts at 3 on new game. Carries across levels and across death; reset to 3 only on new-game-from-title. Decremented by 1 on entering `dying`. Reaching 0 routes `dying → game_over` instead of respawn. Losing a life does not reset `Ammo` (ammo carry-across is unchanged — see [ADR-0007](docs/adr/0007-finite-ammo-manual-fire.md)). No extra-life-from-score mechanic in the first slice (deferred).

- **Track** — one of the 4 music stems (`drums`, `bass`, `lead`, `chords`), each bound to a dot color via `Audio::Manager::DOT_COLORS` (red→bass, green→drums, blue→chords, yellow→lead). Eating a dot of a color advances its track.

- **Track completion** — per-`Track` progress ratio: dots eaten of that color ÷ total dots of that color in the level. Dot-based only (not time/score). Drives the audio progression mix and the HUD's 4 track meters. Backed by a live remaining-by-color counter on `Pellets` (decremented on eat), not a rescan. Reaching 100% (the colour's last dot eaten) awards a flat **1000-point track-completion bonus** (G1), fired per-track the instant the count hits 0 — with a score popup at that dot's cell, a HUD meter flash, and an audio stinger. Unlike `Eat freeze` it does not freeze the world. The popup runs on a `Game`-owned path, not `EatSequencer`.

- **HUD** — screen-space overlay. Shows score (run-long total), `Life` count as a row of player-sprite icons, the `HUD ammo row`, 4 `Enrage gauge`s, and a small `Beat pulse` indicator. Visible in `ready` / `playing` / `dying`; dimmed under a `Pause` overlay; replaced by the `Highscore table` screen in `game_over`.

- **Enrage gauge** — one of the HUD's 4 per-`Territory` bars. Bar fill = territory clearance % (same signal that drives `Enrage` step-ups, so the bar implicitly previews the next step). Left of each bar sits a 32×32 ghost-identity icon (Blinky/Pinky/Inky/Clyde) that carries the discrete state the bar can't: a red overlay at `:enrage1`, a stronger red + beat-synced pulse at `:enrage2`, and a dimmed alpha when the territory is `Pacified`. Replaces the earlier `Track completion` meters (same slots, repurposed per ADR-0010).

- **Beat pulse** — small square HUD indicator centred above the `Enrage gauge`s that pops bright + large on each downbeat and decays to a dim minimum just before the next beat. Visually mirrors the `LEVEL_BPM` (128 BPM ⇒ 28.125 frames/beat). Driven by `args.tick_count` so it stays in phase across pause, identical to `BeatClock`'s tick-derived audio scheduling.

- **Ready** — level-intro `Game state`, entered at level start only (not on respawn). Actors placed and frozen; a beat-synced count-in (≈1 bar at `LEVEL_BPM`) plays metronome clicks, no track. Automatic `ready → playing` at count-in end, no input gate.

- **Dying** — `Game state` entered on `Body-contact`. World frozen (reuse the `EatSequencer.frozen?` pattern). Two phases: (1) fixed-frame death animation (~30–40 frames, not beat-synced) with music ducked out on an ease-out; (2) actors teleport to spawn cells, projectiles cleared, phase scheduler reset, then the `Camera` eases from its current position to the reset player while music eases back in to the current track-completion mix levels. Exits automatically: `dying → playing` when the ease completes if `Life > 0` (decremented on entry), else `dying → game_over`.

- **Game over** — terminal `Game state` reached from `dying` when `Life` hits 0. Shows the UI5 `Highscore table` with final score, level reached, and `Play clock` time formatted `MM:SS`. Two menu options: "retry" (request scene `playing` → full `Game` rebuild) and "back to title" (request scene `title`). Both routes go through the unified `request_scene` path — the only remaining full-rebuild path. New qualifying scores prompt 3-char initials entry before the menu is interactable.

- **Level** — integer difficulty index of the current run, owned by `Game`, starting at 1. Incremented in `start_next_level` (the in-place level loop); reset to 1 only by a full `Game` rebuild (game-over → `request_game_reset`). Drives `LevelConfig` lookups for ghost speed ratios, ghost tunnel speed, Cruise Elroy dot thresholds/speeds, and the scatter/chase phase table — all seeded through a single `Game#apply_level_config` seam called from `initialize` (level 1) and `start_next_level`. Pac-Man's own speed does **not** scale with level: it stays beat-locked at 4 cells/beat regardless (see [ADR-0009](docs/adr/0009-pac-man-speed-stays-beat-locked.md)). Not shown in the `HUD` — the OG communicates level only via the deferred fruit/bonus row, never a number.

- **LevelConfig** — frozen per-`Level` data table (`app/level_config.rb`) ported from the OG *Pac-Man Dossier* Table A.1 ([docs/OG/pacman_dossier_extracts.md](docs/OG/pacman_dossier_extracts.md)). Returns ghost speed/tunnel ratios, Elroy 1/2 dot thresholds and speed ratios, and the scatter/chase phase table for a given level; levels past the last explicit row clamp to it (the dossier's "21+" row). Speed values are **ratios** of `PLAYER_SPEED`, not absolute speeds, keeping the beat the single tempo source of truth. Elroy ratios may exceed 1.0 (faster than Pac-Man) from level 5 on — intended. The Fright. and Bonus columns of Table A.1 are unused (no frightened state per [ADR-0007](docs/adr/0007-finite-ammo-manual-fire.md), no fruit system yet).

- **Territory** — the dot quadrant owned by the ghost that scatters to its corner: red/top-left→Blinky, green/top-right→Pinky, blue/bottom-left→Clyde, yellow/bottom-right→Inky. The mapping falls out of existing scatter-target geometry. Dot colours are recoloured to match the owner ghost's identity colour, and the maze floor carries a subtle per-`Territory` tint. Designed but not yet implemented — see [ADR-0010](docs/adr/0010-quadrant-territory-enrage.md).

- **Enrage** — a ghost's discrete escalation level, keyed to its `Territory`'s clearance %. Monotonic (eaten dots never return). Three steps with stacked effects: scatter-suppression (any non-`:off` step → ignores scatter, keeps chasing), a modest per-`Level` speed bump, and **bullet resistance** — `:off` takes 1 hit, `:enrage1` takes 2, `:enrage2` is **immune** to projectiles. Partial bullet damage clears on step-up. Replaces `Cruise Elroy` — the generalisation of Elroy to all four ghosts, region-scoped — so `LevelConfig`'s Elroy columns re-scope from maze-wide dot counts to per-`Territory` thresholds. The HUD's 4 colour meters are the `Enrage gauge`s (repurposed from `Track completion`). See [ADR-0010](docs/adr/0010-quadrant-territory-enrage.md) and [ADR-0011](docs/adr/0011-enrage-bullet-resistance.md).

- **Pacify** — finishing a `Territory` (its last colour dot eaten, on `Pellets#eat`'s `track_cleared` event) permanently despawns its owner ghost for the rest of the level, plus an `Enrage`-scaled score bonus. The carrot to Enrage's stick; a four-toothed difficulty sawtooth across a level. Persists through player death (clearance isn't reset). `ReleaseSchedule` skips despawned ghosts. Currently a plain despawn; a maze-layout prison cell that visibly traps the ghost is a deferred upgrade. See [ADR-0010](docs/adr/0010-quadrant-territory-enrage.md).

- **Ammo** — integer count of bullets the player currently holds. Starts at 0 on each new level, increases by `AMMO_PER_POWER_PELLET` (= 5) per power-pellet pickup, no cap. Decremented by 1 per fired bullet. Carries across player death; resets to 0 on level complete. See [ADR-0007](docs/adr/0007-finite-ammo-manual-fire.md).

- **Fire input** — edge-triggered key press (Space / controller south) that spends one ammo and spawns a projectile in the player's current travel direction. No-op when ammo is 0 or `player.direction == NONE`. No rate limit beyond edge-trigger.

- **Empty-mag** — fire input pressed with 0 ammo. Silent no-op; HUD already signals 0 via the icon row.

- **Projectile** — bullet spawned by the fire input. Travels at 2× player speed in the firing direction. Walls stop, tunnels wrap. Despawns on wall hit or any non-(eaten, in-house) ghost contact. A frightened/vulnerable ghost variant no longer exists — bullets kill any active ghost via `EatSequencer.on_ghost_eaten`. In-flight count is uncapped; cleared on player death and level complete. See [ADR-0007](docs/adr/0007-finite-ammo-manual-fire.md).

- **Body-contact** — overlap between player rect and any non-house, non-eaten ghost rect. Always fatal to the player. Originally introduced in ADR-0006 and unchanged by ADR-0007.

- **Eat chain** — time-windowed combo bonus owned by `EatSequencer` (G2). Consecutive ghost kills escalate the award through 200 → 400 → 800 → 1600 (capped). Each kill arms a `CHAIN_TIMEOUT_TICKS` window (180 ticks, ~3s); a kill within the window escalates, otherwise the chain resets to 200. Replaces the OG frightened-window chain — frightened state was deleted in [ADR-0007](docs/adr/0007-finite-ammo-manual-fire.md), so the chain is now bounded only by the timer, not by a power-pellet window.

- **Eat freeze** — short pause the game enters when a ghost is killed; popup is shown via `EatSequencer.popup`. Other contacts cannot resolve during the freeze. Retained under ADR-0007; tuning deferred to playtest.

- **HUD ammo row** — bottom-strip readout: up to 5 bullet icons, with a `+` glyph appended when ammo exceeds 5. Always visible.

- **Tunnel** — explicit `t`-marked cell that slows ghosts and wraps movement toroidally on X. See [ADR-0005](docs/adr/0005-explicit-tunnel-tile.md), [ADR-0001](docs/adr/0001-toroidal-maze-x-axis.md).

- **Role** — passability class on a tile (player, ghost, ghost-eaten, ghost-leaving). Drives which actors can enter. See [ADR-0004](docs/adr/0004-passability-roles-via-tiles.md).

- **World space** — authoritative coordinate space for all gameplay, physics, collision and beat-sync. Unit is the cell: `CELL_SIZE` = half the player sprite height (`Player::PLAYER_SPRITE_HEIGHT / 2` = 48 world px). The 64×96 player sprite renders native and unscaled, spanning a 2×2 cell area — a 2-cell tunnel then matches the sprite height exactly. Frozen w.r.t. the camera — no gameplay code knows the camera exists. See [ADR-0008](docs/adr/0008-follow-camera-world-screen-split.md).

- **Screen space** — final pixel space of the 1280×720 output. HUD and popups live here directly. Actors / maze / pellets live in world space and are mapped to screen space by the **Camera**.

- **Sprite Lab** — dev-only `Scene` reachable from `Title`, hidden in production builds via `$gtk.production?`. Selector + stage layout: left list picks entity (player, blinky, pinky, inky, clyde, eaten, …), right list picks state; stage renders the chosen sprite at native size in default facing. Keyboard + mouse navigation. Modifier keys toggle render-affecting state (armor-flash on/off, enrage step cycle). Beat-driven animation at 120 BPM by default, toggleable to free-running tick time; pause + frame-step supported. Silent (no audio). Owns no `Game` instance — uses real `Player` / `Ghost` instances per catalog entry as the sprite source of truth (so procedural anims like player death stay in sync with shipping code).

- **Render-affecting state** — visual modifier on an actor that changes its rendered sprite but is **not** part of its FSM state — e.g. ghost `enrage_step`, `armor_flash_ticks`, `eaten_hit` scale-pulse, player death-anim elapsed. Distinct from `Game state` / ghost FSM state so the `Sprite Lab` enumerates the FSM states on one axis and exposes render-affecting modifiers as orthogonal toggles instead of materialising every combination as a menu item.

- **Camera** — pure view transform (world → screen): a `zoom` factor plus translation. Render-only; never feeds back into physics. Follows the player (hard-lock) so the zoomed-in maze scrolls: Y clamps to maze bounds, X follows freely and draws the world modulo world-width. Fixed `zoom = 1.0`. During `Dying` phase 2 the camera leaves hard-lock and eases (ease-in-out, duration proportional to distance clamped to a min/max frame range, short toroidal path on X) from its current position to the reset player, then snaps back to hard-lock the frame `dying → playing` fires. See [ADR-0008](docs/adr/0008-follow-camera-world-screen-split.md).
